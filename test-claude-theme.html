BitsMoE — Efficient Spectral Energy-Guided Bit Allocation for MoE LLM Quantization

Authors: Jiayu Zhao, Zihan Teng, Minhao Fan et al. · Venue: arXiv preprint, June 2026

BitsMoE addresses the memory bottleneck of Mixture-of-Experts LLMs by decomposing each MoE layer via SVD into a shared basis and expert-specific spectral factors. The shared basis is retained without quantization to preserve common cross-expert structure, while expert-specific factors are quantized using a spectrum-wise mixed-precision scheme where bit-widths are determined by an integer linear program minimizing estimated reconstruction loss under a fixed budget.

Key result: Under 2-bit quantization on Qwen3-30B-A3B-Base, BitsMoE accelerates quantization by 12.3×, improves average accuracy by 27.83 percentage points, and increases decoding speed by 1.76× over GPTQ.

Demo2Reward — From Demonstrations to Rewards via Test-Time VLM Prompt Optimization

Authors: Christian Gumbsch, Leonardo Barcellona, Lennard Schünemann et al. · Venue: arXiv preprint, May 2026

Demo2Reward optimizes the language instruction of a VLM-based reward model using only 3–10 expert demonstration trajectories at test time. It eliminates manual reward engineering by automatically refining prompts to reduce false positives while preserving true positives. No additional model training or computation is required during policy learning. Validated on both simulated robotic tasks and real-world robot learning.

Key result: Consistently outperforms existing zero-shot and few-shot VLM reward models across multiple simulated robotic manipulation tasks and policy backbones; successfully transfers to physical robot policy learning.

Authors: Hao Xu, Rite Bo, Fausto Giunchiglia, Yingji Li, Rui Song · Venue: ACL 2026 Main

DOPA addresses the practical challenge where the target domain is inaccessible during demonstration selection for in-context learning. Math & Physics — Kernel PCA

Day 8 of 11 · Kernel Methods Track

From Linear PCA to Kernel PCA

Classical PCA finds linear directions of maximum variance in the input space $ \mathcal{X} \subseteq \mathbb{R}^d $. But real data is often not linearly separable in the raw feature space. The kernel trick — implicitly mapping data into a high-dimensional (possibly infinite) feature space $ \mathcal{H} $ via $ \phi: \mathcal{X} \to \mathcal{H} $ and performing PCA there — yields Kernel PCA (KPCA), which can capture nonlinear structure without explicitly computing $ \phi $.

Formal Setup

Let $ \{\mathbf{x}_1, \ldots, \mathbf{x}_n\} \subset \mathcal{X} $ be centered in feature space: $ \sum_i \phi(\mathbf{x}_i) = \mathbf{0} $. The sample covariance in $ \mathcal{H} $ is:

\[ \mathbf{C} = \frac{1}{n} \sum_{i=1}^n \phi(\mathbf{x}_i) \phi(\mathbf{x}_i)^\top \]

We seek eigenvectors $ \mathbf{v} \in \mathcal{H} $ satisfying $ \mathbf{C} \mathbf{v} = \lambda \mathbf{v} $. Since $ \mathbf{C} $ is a sum of $ n \) rank-1 outer products, $ \mathbf{v} $ lies in the span of $ \{\phi(\mathbf{x}_i)\} $:

\[ \mathbf{v} = \sum_{i=1}^n \alpha_i \phi(\mathbf{x}_i) \]

Substituting into the eigenvector equation and multiplying both sides by $ \phi(\mathbf{x}_k)^\top $ gives:

\[ \frac{1}{n} \sum_{i=1}^n \alpha_i \sum_{j=1}^n (\phi(\mathbf{x}_k)^\top \phi(\mathbf{x}_j)) (\phi(\mathbf{x}_j)^\top \phi(\mathbf{x}_i)) = \lambda \sum_{i=1}^n \alpha_i (\phi(\mathbf{x}_k)^\top \phi(\mathbf{x}_i)) \]

This is the key step where the kernel trick enters. Define the kernel matrix $ \mathbf{K} \in \mathbb{R}^{n \times n} $ with entries $ K_{ij} = \langle \phi(\mathbf{x}_i), \phi(\mathbf{x}_j) \rangle = k(\mathbf{x}_i, \mathbf{x}_j) $, where $ k $ is a positive-definite kernel. The equation above reduces to:

\[ \mathbf{K}^2 \boldsymbol{\alpha} = n \lambda \mathbf{K} \boldsymbol{\alpha} \]

For $ \mathbf{K} $ invertible, this simplifies to the standard eigenvalue problem:

\[ \mathbf{K} \boldsymbol{\alpha} = n \lambda \boldsymbol{\alpha} \]

Where $ \boldsymbol{\alpha} = (\alpha_1, \ldots, \alpha_n)^\top $ and eigenvalues are $ \lambda_j $ with corresponding $ \boldsymbol{\alpha}^{(j)} $ normalized so that $ \langle \mathbf{v}_j, \mathbf{v}_j \rangle = \lambda_j n \langle \boldsymbol{\alpha}^{(j)}, \boldsymbol{\alpha}^{(j)} \rangle = 1 $.

Projection onto Principal Components

For a test point $ \mathbf{x} $, the projection onto the $ j $-th principal component in $ \mathcal{H} $ is:

\[ \langle \mathbf{v}_j, \phi(\mathbf{x}) \rangle = \sum_{i=1}^n \alpha_i^{(j)} k(\mathbf{x}_i, \mathbf{x}) \]

Notice: we never compute $ \phi(\mathbf{x}) $ explicitly. The entire computation uses only the kernel function $ k $.

Centering in Feature Space

If $ \phi(\mathbf{x}_i) $ are not centered, we must work with the centered kernel matrix:

\[ \tilde{\mathbf{K}} = \mathbf{K} - \mathbf{1}_n \mathbf{K} - \mathbf{K} \mathbf{1}_n + \mathbf{1}_n \mathbf{K} \mathbf{1}_n \]

where $ (\mathbf{1}_n)_{ij} = 1/n $. For a test point, the centered kernel vector $ \tilde{\mathbf{k}}(\mathbf{x}) $ has entries:

\[ \tilde{k}_i(\mathbf{x}) = k(\mathbf{x}_i, \mathbf{x}) - \frac{1}{n} \sum_{r=1}^n k(\mathbf{x}_r, \mathbf{x}) - \frac{1}{n} \sum_{s=1}^n k(\mathbf{x}_i, \mathbf{x}_s) + \frac{1}{n^2} \sum_{r,s} k(\mathbf{x}_r, \mathbf{x}_s) \]

Worked Example 1: RBF Kernel on Concentric Circles

Consider the classic "concentric circles" dataset in $ \mathbb{R}^2 $, which is not linearly separable. Apply KPCA with the RBF kernel $ k(\mathbf{x}, \mathbf{y}) = \exp(-\gamma \|\mathbf{x} - \mathbf{y}\|^2) $ with $ \gamma = 0.1 $.

Step 1: Compute $ \mathbf{K} \in \mathbb{R}^{100 \times 100} $ where $ K_{ij} = \exp(-0.1 \|\mathbf{x}_i - \mathbf{x}_j\|^2) $.

Step 2: Center $ \mathbf{K} $ to get $ \tilde{\mathbf{K}} $.

Step 3: Solve $ \tilde{\mathbf{K}} \boldsymbol{\alpha} = n\lambda \boldsymbol{\alpha} $ for the top 2 eigenvectors.

Step 4: Project each point onto $ \mathbf{v}_1, \mathbf{v}_2 $ via $ \sum_i \alpha_i^{(j)} k(\mathbf{x}_i, \mathbf{x}) $.

The resulting 2D representation separates the circles cleanly — something classical PCA cannot achieve since the variance structure is identical in all radial directions.

Worked Example 2: Preimage Problem

Given a point $ \mathbf{z} $ in feature space $ \mathcal{H} $, find $ \mathbf{x} \in \mathcal{X} $ such that $ \phi(\mathbf{x}) \approx \mathbf{z} $ (needed for denoising). For the RBF kernel, this reduces to minimizing:

\[ \|\mathbf{z} - \phi(\mathbf{x})\|^2 = \langle \mathbf{z}, \mathbf{z} \rangle - 2 \sum_{i=1}^n \beta_i k(\mathbf{x}_i, \mathbf{x}) + k(\mathbf{x}, \mathbf{x}) \]

where $ \beta_i $ are the expansion coefficients of $ \mathbf{z} $. For the RBF kernel, $ k(\mathbf{x}, \mathbf{x}) = 1 $, and the minimization becomes an unconstrained nonlinear optimization over $ \mathbf{x} $ solvable via gradient descent.

What Comes Next: The Representer Theorem

KPCA is a special case of a deeper principle: any solution to a regularized empirical risk minimization with a reproducing kernel Hilbert space norm penalty takes the form $ f(\mathbf{x}) = \sum_i \alpha_i k(\mathbf{x}_i, \mathbf{x}) $. This Representer Theorem underlies kernel ridge regression, kernel SVMs, and Gaussian processes — connecting PCA to the entire kernel methods family.

ML Deep Dive — Deep Double Descent

4 min read
Day 8 of 11 · Modern Deep Learning Theory

The Bias-Variance Tradeoff, Revisited

The classical bias-variance tradeoff from statistical learning theory states that as model complexity increases, bias decreases monotonically while variance increases monotonically. The optimal model sits at the U-shaped minimum of test error, just before variance overwhelms bias. This story applies to well-specified, low-capacity models like linear regression with $ p < n $.

Modern neural networks shatter this narrative. They operate in the overparameterized regime ($ p \gg n $) where the classical curve would predict catastrophic variance — yet they generalize well. Worse: test error exhibits a second peak at the interpolation threshold ($ p = n $ for linear models), then decreases as model size increases further. This is deep double descent.

Formal Definition

Let $ f_{\theta}: \mathcal{X} \to \mathcal{Y} $ be a model with $ p $ parameters trained on $ n $ samples. The test error (risk) $ R(p, n) $ exhibits two distinct regimes:

\[ R(p, n) = \begin{cases} \text{U-shaped (classical)} & p < n \\ \text{divergent peak} & p \approx n \quad (\text{interpolation threshold}) \\ \text{decreasing} & p > n \quad (\text{overparameterized}) \end{cases} \]

For a linear model $ f_{\mathbf{w}}(\mathbf{x}) = \mathbf{w}^\top \mathbf{x} $ with $ \mathbf{X} \in \mathbb{R}^{n \times p} $, the minimum-norm least-squares solution (interpolating solution) is:

\[ \hat{\mathbf{w}} = \mathbf{X}^\top (\mathbf{X} \mathbf{X}^\top)^{-1} \mathbf{y} \]

This solution exists precisely when $ \mathbf{X} \mathbf{X}^\top $ is invertible, which requires $ p \geq n $. At $ p = n $ (assuming full rank), the matrix is exactly square and the solution is unique — it perfectly interpolates training data but typically has enormous $ \ell_2 $ norm, which translates to high variance for out-of-sample predictions. As $ p $ increases beyond $ n $, the solution manifold expands and we can select solutions with smaller norm, reducing variance more than the increase in bias.

Derivation: Risk Decomposition at Interpolation

Consider the random design setting $ \mathbf{x}_i \sim \mathcal{N}(0, \mathbf{I}_p) $, $ y_i = \mathbf{w}_*^\top \mathbf{x}_i + \varepsilon_i $ with $ \varepsilon_i \sim \mathcal{N}(0, \sigma^2) $. The minimum-norm interpolator has risk:

\[ R(p, n) = \sigma^2 \cdot \frac{p}{p - n - 1} + \|\mathbf{w}_*\|^2 \cdot \frac{n}{p - n - 1} \]

for $ p > n + 1 $ (a result due to Hastie et al., 2019). The key observation: as $ p \to n+1 $ from above, both terms diverge to $ \infty $ — this is the peak. As $ p \to \infty $, the first term tends to $ \sigma^2 $ (the irreducible error) and the second term tends to $ 0 $ (the bias vanishes). So infinite overparameterization recovers the Bayes-optimal risk.

Historical Context

Belkin et al. (2019) first clearly articulated the "double descent" phenomenon in "Reconciling modern machine-learning practice and the classical bias-variance trade-off". The phenomenon had been observed earlier by Opper (1995) in the context of Gaussian process regression, but was dismissed as a theoretical curiosity. Modern understanding emerged from the confluence of three threads:

  1. Random matrix theory (Hastie et al., 2019): exact risk expressions for linear models with random features
  2. Neural tangent kernel (Jacot et al., 2018): infinite-width networks behave as kernel machines, inheriting the same double descent
  3. Empirical deep learning (Nakkiran et al., 2019): demonstrated double descent for ResNets, CNNs, and Transformers on real data (CIFAR-10, ImageNet)

Failure Modes and Practical Implications

The double descent peak has important practical consequences. Training exactly at the interpolation threshold (e.g., a model whose parameter count matches the dataset size) produces the worst possible test error. For modern overparameterized networks, this is invisible because $ p \gg n $ from the start. However, double descent can be observed along other axes of model complexity: training epochs (epoch-wise double descent) and dataset size (sample non-monotonicity).

Epoch-wise double descent: For a fixed $ p \gg n $, test error can initially decrease, then increase (as the model reaches the interpolation threshold in function space), then decrease again. This explains why early stopping can sometimes help even in massively overparameterized regimes — it halts before the interpolation peak.

Connection to NTK

Recall Day 7's NTK analysis: in the infinite-width limit, a neural network $ f(\mathbf{x}; \theta) $ evolves under gradient flow as a linearized model:

\[ f_t(\mathbf{x}) \approx f_0(\mathbf{x}) - \Theta_t(\mathbf{x}, \mathcal{X}) \Theta(\mathcal{X}, \mathcal{X})^{-1} (\mathbf{I} - e^{-\eta \Theta(\mathcal{X}, \mathcal{X}) t / n}) (\mathbf{y} - f_0(\mathcal{X})) \]

At convergence ($ t \to \infty $), the NTK-ridge-regression solution inherits the same double descent behavior as the kernel matrix $ \Theta(\mathcal{X}, \mathcal{X}) $ approaches rank-deficiency at the interpolation threshold. The key difference: NTK kernels behave like $ p \to \infty $ always, so the model lives firmly in the overparameterized regime after convergence — but early in training, the effective number of "active" features can be much smaller, creating conditions for epoch-wise double descent.

Today's Topic — AI Safety & Alignment

3 min read

This week, AI safety and alignment moved from abstract research conference debates to the center of U.S. national security policy — driven by three intersecting events that together define a new regulatory paradigm.

The Anthropic Export Control Precedent

The U.S. government's directive to block foreign access to Anthropic's Mythos 5 and Fable 5 is unprecedented in scope. It is not a voluntary opt-in under the new EO framework, but a binding export control order under Commerce Department authority — effectively treating a frontier AI model like a missile guidance system. The trigger was a claimed jailbreak by an unnamed third party, highlighting a fundamental gap in current safety evaluation: there exists no standardized, government-accepted protocol for determining when a model is "safe enough" for unrestricted deployment. The administration's response was an all-or-nothing export ban because the evaluation infrastructure to make a more nuanced determination does not yet exist.

This parallels the core insight from the International AI Safety Report 2026 (published February 2026): existing safety evaluations focus on narrow, known vulnerability categories, but frontier models exhibit emergent capabilities that evade these tests. The report recommended scenario-based evaluations and adversarial testing as necessary complements to standardized benchmarks.

The Voluntary Framework Paradox

The June 2 Executive Order creates a voluntary framework for pre-release government access to frontier models, while explicitly disclaiming mandatory licensing. Within ten days, the government used its existing export control authority to force exactly that outcome for Anthropic. This creates a paradoxical incentive: companies that participate in the voluntary framework gain a transparent, collaborative relationship with government evaluators; companies that release models without engagement risk sudden, disruptive action via export controls or other existing authorities. The framework is "voluntary" only in the sense that companies can choose when and how to structure government access, not whether government access will ultimately be demanded.

From an alignment research perspective, this highlights a gap between technical alignment (ensuring models do what their operators intend) and regulatory alignment (ensuring models meet government-defined safety thresholds). Anthropic's Fable 5 safety routing mechanism — falling back to a less capable model on high-risk queries — is a technical alignment solution, but it failed to prevent the regulatory outcome because the concern was not model behavior but model access by foreign actors.

Recent Alignment Research

Several recent papers articulate the technical challenges exposed by these policy developments:

  • "Position: AI Safety Requires Effective Controllability" (arXiv:2605.27117) argues that alignment is insufficient without explicit mechanisms for human operators to halt, modify, or inspect model behavior in deployment. The Anthropic case confirms this: the "control" mechanism applied was not technical but legal (export controls), applied externally rather than built into the model.
  • "AI Alignment Breaks at the Edge" (arXiv:2602.20042) shows that current alignment techniques (RLHF, constitutional AI) degrade significantly on out-of-distribution inputs. This directly relates to the PGDR research program: worst-group accuracy in alignment systems is a practical national security concern when an adversary deliberately seeks edge cases.
  • "What Matters For Safety Alignment?" (arXiv:2601.03868) systematically evaluates which training interventions (data curation, RLHF scaling, red teaming) most improve safety metrics, finding that RLHF scaling alone accounts for ~60% of safety gains, with data quality contributing the remainder.

The week's events establish a new reality: alignment is no longer a purely technical problem solved before deployment. It is an ongoing governance challenge where technical safety mechanisms, regulatory frameworks, and geopolitical competition intersect. For the PGDR research program, the lesson is direct — robustness to distribution shift is not a nice-to-have academic property; it is the difference between a model that can be deployed confidently and one that triggers an export ban.
